 # [재활운동어플리케이션 – 동네 기반 재활 헬스장 & 운동 코스 추천]

---

## 1. Overview

### Problem Statement

- **문제 정의**  
  - 서울/경기 거주자 중, 스스로 움직일 수 있고 헬스장도 이용하지만  
    “재활운동은 어떻게, 어디서, 무엇을 해야 할지” 모르는 사람들이 많다.  
  - 구체적으로:
    - 허리/어깨/무릎 등이 불편해 재활운동이 필요하지만  
      ① 어떤 운동을 해야 하는지 모르고  
      ② 동네에서 재활에 적합한 헬스장을 찾기도 어렵고  
      ③ 병원·센터 기반 재활은 시간·비용 부담이 크다.
  - 기존 대형 포털(네이버/카카오맵) 지도에서 헬스장 검색은 가능하지만  
    - 조용한지, 재활운동하기 적합한지, PT/재활코치 유무, 평소 혼잡도 등  
      재활 목적에 꼭 필요한 정보는 찾기 힘들다.

- **왜 중요한가**  
  - 재활운동은 꾸준함이 핵심인데,  
    “어디서 / 어떻게”가 막히면 시작조차 못하는 경우가 많다.  
  - 서울/경기에는 헬스장/필라테스/소규모 PT샵이 매우 많지만,  
    **재활에 특화된 정보**가 구조화되어 있지 않아 수요-공급 미스매치가 발생한다.  
  - 스스로 운동을 해보고 싶은 사람에게  
    “동네에서 갈 수 있는 곳 + 오늘 당장 할 수 있는 루틴”이 같이 제공되면  
    실제 행동 전환(시도)이 크게 높아질 수 있다.

- **타겟 고객**  
  - 연령대: 전 연령층(10대 후반~60대), 특히 20–40대 비중 높음  
  - 직업: 직장인, 프리랜서, 대학생 등  
  - 위치: 서울/경기 (MVP는 강남/홍대 인근 지역부터)  
  - 행동 패턴:
    - 주 2회 이상 헬스장/운동시설 이용, 1~2시간 이상 머무름  
    - 시끄럽고 붐비는 곳보다 비교적 조용한 환경 선호  
    - 허리/어깨/무릎 등 특정 부위 불편함 경험 있음  
    - 병원 재활까지는 아니더라도, “통증 악화는 막고 싶다”는 니즈 존재

- **전략적 적합성**  
  - 1인 개발자로서, 대형 플랫폼이 커버하지 않는 **“재활 친화 헬스장 + 초보 재활 루틴”**라는 틈새를 공략할 수 있다.  
  - 지도/로컬 기반 서비스, 챗봇 플로우, 운동 코스 추천 등  
    재사용 가능한 기술/경험(포트폴리오)을 쌓을 수 있다.  
  - 향후 재활 코치/트레이너, 재활 특화 센터와의 협업/수익 모델(광고, 제휴, 예약 중개 등)로 확장 가능성이 있다.

- **핵심 전제/가정**  
  - 사용자는 이미 어느 정도 운동 경험이 있고, 스스로 기구를 사용할 수 있다.  
  - 정확한 ‘의학적 진단’보다,  
    “어느 부위가 불편한지, 어떤 동작이 힘든지” 수준의 자가 문진 + 기본적인 재활운동 루틴만으로도 큰 가치를 느낀다.  
  - 서울/경기 내에 “조용하고, 재활운동에 적합한 헬스장/공간”이 충분히 존재하며,  
    온라인에서 제대로 정리되지 않았을 뿐이다.  
  - 카카오맵 무료 플랜과 1인 개발로, 3개월 안에 **웹/모바일에서 사용 가능한 MVP**를 구현할 수 있다.

### Proposed Work

- **솔루션 개요 (1–2문장)**  
  동네 기반으로 재활운동하기 좋은 헬스장/공간을 추천해주고,  
  채팅형 문답을 통해 사용자의 불편한 부위를 파악해 **약 1시간 30분짜리 재활 운동 코스**를 자동 생성해주는 서비스.

- **핵심 기능 (3–5개)**  
  1. **위치 기반 주변 재활 헬스장 검색 (1km 내)**  
     - 현재 위치/선택 위치 기준 반경 1km 내 재활 친화 헬스장/운동공간 리스트 및 지도 표시  
  2. **실용 정보 필터링**  
     - 조용한 분위기, 재활 기구 구비, PT/재활 코치 여부, 샤워실/주차, 가격대 등 필터  
  3. **(경량) 혼잡도 정보 제공**  
     - “한산/보통/붐빔” 수준의 예상 혼잡도 (시간대 기반 + 사용자/점주 업데이트)  
  4. **주민 리뷰 – 태그 기반 시스템**  
     - 별점 대신 “조용함”, “기구 상태 좋음”, “재활 친화”, “복잡함” 등의 태그 선택형 리뷰 + 짧은 코멘트  
  5. **채팅형 재활 코스 추천**  
     - 챗봇처럼 질문(불편 부위, 통증 정도, 사용 가능한 기구, 운동 경험 등)에 답하면  
       약 1시간 30분 분량의 재활 운동 루틴(준비운동–메인–마무리 스트레칭)을 자동 생성

- **사용자 플로우**

  1. **입장 & 온보딩**  
     - 웹/앱 접속 → 현재 위치 사용 허용 or 동네 검색(예: 강남역, 홍대입구역)
  
  2. **재활 헬스장 찾기 플로우**  
     - 지도 화면: 내 주변 1km 내 후보 리스트 표시  
     - 상단 필터: “조용한 곳”, “재활기구 있음”, “주차 가능” 등 선택  
     - 각 장소 상세 페이지:  
       - 기본 정보(주소, 가격대, 운영시간)  
       - 태그 리뷰 모음  
       - 혼잡도(한산/보통/붐빔) 표시  
       - 저장(즐겨찾기) 기능
  
  3. **재활운동 코스 추천 플로우**  
     - 홈에서 “내 몸 상태로 운동코스 만들기” 버튼 클릭  
     - 채팅형 문답:
       - 어떤 부위가 가장 불편한가? (허리/어깨/무릎/목/기타)  
       - 어느 정도 통증인가? (1–5 단계)  
       - 사용 가능한 기구? (덤벨/머신/짐볼/매트만 등)  
       - 평소 운동 빈도?  
     - 답변 결과 기반으로 90분 루틴 생성:  
       - 준비운동: 10–20분  
       - 메인 재활운동: 50–60분 (세트·반복, 휴식 시간 포함)  
       - 마무리 스트레칭: 10–20분  
     - 루틴을 저장하고, 근처 재활 헬스장을 바로 연결해 보여줌 (“이 루틴 하기 좋은 근처 헬스장 보기”)

  4. **방문 후 리뷰 작성**  
     - 방문한 헬스장 상세 페이지 → “리뷰 남기기”  
     - 태그 선택(조용함/복잡함/재활에 좋음/장비 오래됨 등) + 50자 내외 코멘트  
     - 리뷰는 다른 사용자의 필터링·의사결정에 활용

## 2. Decision / Risk

### Stakeholders

- **직접 이해관계자**
  - 재활운동이 필요하지만, 동네에서 스스로 해결하고 싶은 일반 사용자  
  - 재활 친화적 헬스장/소규모 PT샵/운동공간 운영자  
  - 제품 기획 및 개발을 모두 담당하는 1인 개발자(본인)

- **간접 이해관계자**
  - 재활 코치, 물리치료사, 트레이너 (루틴 설계 자문 역할)  
  - 온라인 커뮤니티(헬스/재활/자세교정 카페, 디시·블라인드 등) – 초기 유입·리뷰 소스  
  - 향후 제휴 가능 파트너: 재활 센터, 병원, 보험사 등

---

### Risk

- **주요 위험 요소 / 발생 가능성 / 영향도 / 대응 방안**

1. **초기 리뷰 부족(cold start 문제)**  
   - 가능성: 상 / 영향도: 중~상  
   - 대응:
     - 강남/홍대 일부 소수 헬스장부터 ‘큐레이션 리스트’ 형태로 시작  
     - 지인·커뮤니티를 통해 초기 50~100개 태그 리뷰를 직접 확보  
     - MVP 단계에서는 “에디터 추천(운영자 코멘트)” 섹션으로 리뷰 부족 보완

2. **장소 정보 업데이트 부담 (1인 개발자의 수동 입력)**  
   - 가능성: 상 / 영향도: 중  
   - 대응:
     - 강남/홍대의 선정된 30~50개 장소를 우선 수동 등록  
     - 스프레드시트 + 간단한 어드민 페이지로 정보 수정/관리  
     - 장소 추가 제안 기능(사용자가 제보 → 운영자가 승인 후 등록)

3. **재활운동 코스 추천의 안전성·신뢰도**  
   - 가능성: 중 / 영향도: 상  
   - 대응:
     - “의료적 진단이나 치료가 아닌, 일반적인 운동 정보”라는 점을 명확히 고지  
     - 고위험 동작 및 통증 악화 가능성이 큰 고강도 운동 제외  
     - 1~2명 이상의 재활/트레이닝 경험자 자문을 받아 기본 템플릿 설계  
     - 통증 심화 시 즉시 중단 및 전문의 상담 안내 문구 반복 노출

4. **카카오맵 API 무료 플랜 한도 초과**  
   - 가능성: 중 / 영향도: 중  
   - 대응:
     - 강남/홍대 지역만 서비스 → 호출량 제한  
     - 캐싱 전략(서버에서 장소 리스트 캐시, 클라이언트 지도 이동 제한)  
     - 일일 호출량 모니터링을 위한 간단한 로깅 구현

5. **사용자 유입 부족 (MAU 300 달성 실패 가능성)**  
   - 가능성: 중~상 / 영향도: 상  
   - 대응:
     - 초기에는 특정 온라인 커뮤니티 2~3곳에 집중 마케팅 (헬스/재활/자세 교정 관련)  
     - “강남 조용한 재활 헬스장 지도”, “홍대 헬스장 혼잡도 지도” 등 니치 키워드 블로그/노션 페이지 제작  
     - 지인 네트워크를 통한 사용·피드백 요청

### Dependencies

- **외부 서비스**
  - 카카오맵 API (지도 표시, 좌표 변환 등 – 무료 플랜)  
  - 호스팅/배포: Vercel, Netlify 또는 저비용 VPS  
  - Analytics/로그: 무료/저가 툴 (Google Analytics, Logtail 등)

- **필요 데이터**
  - 헬스장/운동공간 기본 정보: 이름, 주소, 좌표, 운영시간, 가격대, 시설 정보(샤워/주차/기구 등)  
  - 사용자 리뷰 데이터: 태그, 짧은 코멘트, 작성 일시  
  - 재활운동 루틴 템플릿:  
    - 부위별(허리/어깨/무릎/목/전신),  
    - 강도별/시간 구분된 운동 리스트(운동명, 세트/횟수, 시간, 주의사항)

- **필요 인력**
  - 개발: 1인 (본인) – 프론트엔드 + 백엔드 + 인프라  
  - (파트타임/지인) 재활/트레이닝 자문 1–2명 – 루틴 검토  
  - (선택) 디자인 도움 1명 – UI킷/로고/컬러가이드 (가능하다면)

## 3. Success Criteria

### 핵심 지표 (2–3개)

1. **지표명: 등록 재활운동 & 헬스장 수**  
   - 측정 방법: DB에 활성 상태로 등록된 장소 개수(중복 제외)  
   - 목표 수치: 50개  
   - 달성 기한: 런칭 후 3개월 이내

2. **지표명: 리뷰(태그 리뷰) 수**  
   - 측정 방법: 사용자/운영자 작성 리뷰(태그형) 총 개수  
   - 목표 수치: 200개 이상  
   - 달성 기한: 런칭 후 3개월 이내

3. **지표명: 월간 활성 사용자(MAU)**  
   - 측정 방법: 최근 30일 내 1회 이상 지도 검색 또는 코스 생성을 한 고유 사용자 수  
   - 목표 수치: 300명  
   - 달성 기한: 런칭 후 3개월 이내

### 보조 지표

- 재활 코스 생성 수(누적/월간)  
- 코스 생성 후 지도 검색으로 이어지는 전환율  
- 장소 상세 페이지에서 리뷰 작성을 완료한 비율  
- 최초 방문 후 2주 이내 재방문율

## 4. Definition of Done

### 필수 조건 (Done – MVP 출시 필수)

1. **서비스 기본 기능**
   - 현재 위치/검색 위치 기준 반경 1km 내 장소 리스트 및 지도 표시  
   - 장소 상세 페이지: 기본 정보 + 태그 리뷰 목록 표시  
   - 필터 기능: 최소 3개 이상 (예: 조용한 곳, 주차 가능, 재활기구 있음)  
   - 리뷰 작성 기능: 태그 선택 + 코멘트(선택)

2. **재활 코스 추천**
   - 채팅형 문답 UI (최소 3~5개 질문 플로우)  
   - 부위별·기구별 기본 템플릿 보유  
   - 약 1시간 30분짜리 루틴(준비–메인–마무리) 텍스트 생성 및 화면 표시  
   - “이 코스 하기 좋은 주변 헬스장 보기” 버튼으로 지도 연동

3. **기술·인프라**
   - 모바일 웹 기준으로 반응형 레이아웃 (스마트폰 사용 최적화)  
   - 카카오맵 API 연동 및 기본 성능 확보(스크롤/줌 시 치명적 버그 없음)  
   - 기본적인 에러 로깅/모니터링 (콘솔·간단 서버 로그 수준)

4. **법/정책**
   - 개인정보 처리방침 및 이용약관 페이지 노출  
   - 재활 코스에 “의료적 진단·치료가 아닌 일반 운동 정보”라는 안내 문구 필수 노출

---

### 개선 필요 (but 출시 가능)

- 로그인/회원 시스템 없이, 쿠키/로컬스토리지 기반 가벼운 상태 저장  
- 복잡한 혼잡도 알고리즘 대신, “최근 1시간 내 사용자/점주의 수동 입력” 기반 3단계 표시  
- 디자인 최소화(기본 UI라이브러리 사용, 로고/브랜딩 심플하게)  
- 장소 검색 범위(1km 고정, 세밀한 반경 조절은 후순위)

---

### 치명적 (출시 불가)

- 지도/리스트에서 장소가 제대로 노출되지 않거나, 클릭 시 상세 정보가 보이지 않는 경우  
- 재활 코스 추천 플로우에서 에러로 인해 루틴이 생성되지 않거나, 잘못된(엉뚱한) 내용이 노출되는 경우  
- 약관/안내 없이 “진단/치료”처럼 오인될 만한 표현 사용  
- 리뷰 작성 시 데이터가 유실되거나, 다른 사용자 리뷰가 잘못 덮어써지는 데이터 무결성 문제

---

## 5. Nope (하지 않을 것)

1. **기능명: 실제 전문가(의사/물리치료사)와의 1:1 실시간 채팅/상담**  
   - 이유:  
     - 의료법/원격의료 규제 리스크 크고, 인건비 및 운영 리소스 필요  
   - 대안:  
     - 챗봇 기반 자가 문진 + 사전 정의된 FAQ/가이드 제공  
   - 재고려 시점:  
     - 사용자/트래픽이 일정 수준 이상 확보되고, 협업 가능한 전문가 네트워크가 생긴 이후 (v3+)

2. **기능명: 예약/결제 시스템(헬스장 이용권, PT 예약/결제)**  
   - 이유:  
     - 복잡한 결제 플로우, 취소/환불, 세금 이슈 등 초기 1인 개발 범위를 초과  
   - 대안:  
     - 각 헬스장 상세 페이지에 전화/카톡/웹사이트 링크 노출로 연결만 제공  
   - 재고려 시점:  
     - 특정 헬스장/센터와 제휴 요청이 생기고, 수익 모델 검증 필요 시 (v2 이후)

3. **기능명: 진짜 의미의 실시간 혼잡도(게이트 카운터, CCTV, 기기 연동 등)**  
   - 이유:  
     - 센서/하드웨어/헬스장 시스템 연동 필요 → 비용·협의 난이도 높음  
   - 대안:  
     - 시간대 기반 “예상 혼잡도” + 사용자/점주 수동 업데이트 버튼  
   - 재고려 시점:  
     - 특정 제휴 헬스장과의 깊은 파트너십 체결 시 PoC 형태 (v3+)

4. **기능명: 풀스케일 네이티브 앱(iOS/Android) 동시 개발**  
   - 이유:  
     - 1인 개발, 3개월 안에 안정적인 웹/MVP를 만드는 것이 우선  
   - 대안:  
     - 모바일 웹 또는 PWA 기반으로 시작  
   - 재고려 시점:  
     - MAU 1,000명 이상, 재방문율/체류시간이 의미 있게 나올 때 (v2~v3)

5. **기능명: AI 포즈 인식/카메라 기반 운동 자세 교정**  
   - 이유:  
     - 머신러닝/컴퓨터 비전 개발 리소스와 검증 시간이 많이 필요  
   - 대안:  
     - 텍스트/이미지 기반의 자세 안내, 자가 체크리스트 제공  
   - 재고려 시점:  
     - 충분한 사용자·운동 데이터 확보 후, 고도화 기능으로 (v3+)

## 6. Post-Launch Plan

### v1.1 (Quick Wins – 출시 후 1–2주)

- 리뷰 작성 UX 개선  
  - 자주 쓰이는 태그를 상단에 노출, 탭 한 번으로 리뷰 완료  
  - “코스 수행 후 리뷰 남기기” 푸시/팝업 추가(웹 알림 or 인앱 배너)
- 코스 추천 튜닝  
  - 실제 사용자 피드백을 받아, 과도하게 길거나 어려운 루틴 조정  
  - 같은 부위라도 “헬린이/경험자” 옵션으로 두 가지 난이도 제공
- 기본 SEO/공유 기능  
  - 특정 동네(강남/홍대) 지도·리스트 페이지를 소셜/커뮤니티에 공유하기 쉽게 메타태그 정리

### v2 (Major Features – 다음 분기)

- **장소·운동 데이터 확장**
  - 서비스 지역을 강남/홍대 → 서울 주요 상권(신촌, 건대, 잠실 등)으로 확대  
  - 재활 루틴 템플릿에 부위/강도/시간 세분화
- **헬스장/운동공간 운영자 기능(라이트 버전)**
  - 점주용 장소 정보 수정 요청 폼 (운영시간 변경, 가격, 시설 정보 등)  
  - 혼잡도/이벤트(체험권, PT 할인) 수동 업데이트 기능
- **사용자 계정/즐겨찾기**
  - 간단한 이메일/소셜 로그인 추가  
  - 즐겨찾는 헬스장/코스 저장 & 히스토리 조회 기능

### v3+ (Vision – 장기 방향)

- **정교한 개인 맞춤 재활 코치**
  - 누적 코스 수행 이력, 통증 부위 변화, 선호 환경에 따른 맞춤 코스 추천  
  - “오늘은 상체 휴식, 하체 위주 재활 추천” 등 주기적 제안
- **전문가·제휴 확장**
  - 재활 PT 센터, 소규모 스튜디오와 제휴 → “공식 재활 파트너” 라벨 부여, 노출 우선권  
  - 전문가가 직접 만든 프리미엄 코스(영상/상세 가이드) 유료 판매 가능성 검토
- **서비스 영역 확장**
  - 서울 전역 → 수도권 → 주요 광역시 확장  
  - 재활 중심에서, 자세 교정·사무직/학생 대상 통증 예방 루틴 등으로 확장  
- **수익 모델 다각화**
  - 제휴 헬스장 스폰서 배너/추천 슬롯  
  - 재활용 소도구(폼롤러, 밴드 등) 커머스 연계  
  - 기업 복지/단체 회원 대상 로컬 재활 지도 서비스

---

이 PRD는 3개월, 1인 개발, 강남/홍대 중심이라는 현실적인 제약을 전제로 한 **MVP 설계안**입니다.  
다음 단계로는:

1. 강남/홍대 기준 후보 헬스장 20~30곳 리스팅  
2. 재활 루틴 템플릿(부위별 2~3개씩) 초안 작성  
3. 예상 화면 와이어프레임(지도/상세/채팅/코스 결과) 정의  

까지 정리하면, 실제 개발 착수 범위와 우선순위가 훨씬 선명해질 것입니다.
